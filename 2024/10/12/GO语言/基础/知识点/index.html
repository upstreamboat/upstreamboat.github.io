知识点

知识点

  1. 自动推断变量类型

  2. 全局变量：全包可见；存在未使用的全局变量时可编译通过
    局部变量：函数内可见；存在未使用的局部变量时编译报错

  3. 非引用类型的变量，声明后便会分配内存，可以直接使用，默认自动初始化为类型零值。引用类型不可直接用，必须初始化：

    var m map[int]int
    m[1] = 1 // error：assignment to entry in nil map

  4. 初始化声明只可以用在函数中

  5. 声明时变量类型不可省略，初始化变量类型时可省略

  6. 未使用变量为错误

  7. fmt.Sprintf：有返回值的格式化打印

  8. 运算：

    返回计算结果：+、-、*、/、%
    返回 true/false：==、>、<、>=、<=、!=
    返回 true/false：&&（同真则真）、||（有真则真）、！（非）
    返回计算结果：&（同 1 则 1）、|（有 1 则 1）、^（同 0 异 1）、<<（等效于2^n，高位丢弃、低位补 0）、>>

  9. 位移：y << n == y * (2^n)，左移 n 位就是乘以 2 的 n 次方，高位丢弃，低位补 0；右移 n 位就是除以 2 的 n 次方

  10. iota：自增计数器，遇到 const 声明时重置

  11. Go 语言没有三目运算符，所以不支持  ?:  形式的条件判断

  12. 指针未初始化，无法获取值

  13. 值传递不会影响到实际参数，引用传递会影响到实际参数

  14. 值接收者方法(c client)会将结构体值拷贝一份，然后对副本进行操作，而指针接收者方法(c *client)则直接操作原始结构体指针所指向的对象。

  15. 函数和方法：

    函数：func myFun()

    直接通过包名调用：包名.myFun()

方法：func (c *client) myFun()

需要通过结构体对象调用：c.myFun()
作用：自我总结：1）用来实现接口；2）限制作用域，使其只能被限定对象调用

  1. 匿名函数也是需要调用才会执行，且调用时只执行当前匿名函数，不会执行外部函数

  2. 数组是相同数据类型的集合，结构体是不同数据类型的集合

  3. 使用 go 关键字创建 goroutine 时，被调用函数的返回值会被忽略。

  4. goroutine 间内存共享

  5. 通道的收发操作在不同的两个 goroutine 间进行. 由于通道的数据在没有接收方处理时，数据发送方会持续阻塞，因此通道的接收必定在另外一个 goroutine 中进行

  6. 数据类型：

    1. 本质：固定内存大小的别名
    2. 作用：编译器预算对象(变量)分配的内存空间大小
    3. 分类：
      基本类型：数字、字符串和布尔值
      聚合类型：数组和结构体
      引用类型：切片、映射、通道、指针、函数、接口

      // 对于引用类型的变量，存储的是内存地址，不光要声明它，还要为它分配内存空间
      // 引用类型的零值为 nil
      // 引用类型是指针传递，特别注意：切片是指针传递（会修改底层数组，因为切片是指向数组的指针）、数组是值传递

  7. 当标识符（包括常量、变量、类型、函数名、结构字段等等）以一个大写字母开头，则包外可见，类似 public；如以小写字母开头，则仅包内可见，类似 protected

  8. 文件/目录权限：

    文件：-rwxrwxrwx，目录：drwxrwxrwx（-代表文件，d 代表目录）
    rwx：读、写、执行权限
    -rwx——：文件拥有者的权限；—-rwx—：文件拥有者所在的用户组的权限；——-rwx：其他所有人的权限
    -rwxrwxrwx，对应 0777；-rw-rwxrwx，对应 0677；-r–rwxrwx，对应 0477；—-rwxrwx，对应 0077

  9. Chanel：当通道被关闭时，写入通道会引发 panic，读取通道会一直都是类型零值。

    判断通道是否被关闭的两种方式：

    方式一：i, ok := <-ch1 // 通道关闭后再取值 ok=false
    方式二：for i := range ch2 {fmt.Println(i)} // 通道关闭后会退出 for range 循环

  10. 反射：反射的核心是两个对象，分别是  reflect.Type  和  **reflect.Value**。 它们分别代表了 go 语言中的类型和值。我们可以通过  reflect.TypeOf  和  reflect.ValueOf  来获取到一个变量的类型和值。

    通过反射可以获取目标的类型、值等，也可以通过反射修改其值，但是修改时要注意：

    1）如对结构体中字段修改，字段需可见（即首字母需要大写）
    2）当 reflect.ValueOf(目标)时，目标必须是地址。可以调用 elem()返回“地址 Value”指向的“值 Value”。
    有些方法，是“地址 Type”或“地址 Value”无法访问的，必须转成“值”。

  11. 标签 tag：mapstructure:"port" json:"port" yaml:"port" binding:"required" valid:"matches([0-9])"

    将数据与对象进行关联，可以进行序列化和反序列化。如：

    mapstructure：将配置数据读到结构体字段中
    json：将字段内容写入 json
    yaml：将字段内容写入 yaml 文件
    gorm:"column:name"：将字段与数据库列名对应需要

binding 会在 ShouldBindJSON()时进行检查；valid 会在 govalidator.ValidateStruct()时进行检查

  1. Mutex.TryLock：允许在非阻塞模式下获取锁，如果锁已被获取，该函数将直接返回布尔值  false，而不是一直等待锁释放。

  2. 切片 slice：
    切片是引用类型，是指向底层数组的指针，对切片的操作其实是对底层数组的操作。
    当发生扩容时，该指针地址便会改变，因为扩容是把当前数组内容复制到一个新数组中。

  3. map：value, ok := x[“key”] # map 可以返回两个值，值 2 用来判断是否存在 key

    # 不可以用值 1 判断是否存在 key，因为如果值 1 为类型零值，可能是 key 存在，但其值就是零值

  4. unsafe.Pointer：任意类型的指针。

    一般指针定义时需要指明类型，如*int，而 Pointer 可以和任意类型的指针（或者和 uintptr）相互转换。
    uintptr：类型是一个整型，用来存储十进制的地址。指针无法进行算术运算，转换为 uintptr 后便可以了，因为它是整型。

  5. 无法取地址：常量、字面量、map 中的元素

  6. map、slice、function 是不可比较类型，其余都是可比较类型。如果要比较结构体、数组，其中的元素必须是可比较类型。
    结构体比较规则：

    1. 属性类型、个数、顺序三者都相同；
    2. 属性中不含有不可以比较的类型。即如果含有 map、slice，则结构体不能用==比较。

  7. 类型转换

    1. 数字：强转：float32(num1)
    2. 字符串：strconv.Atoi(“1”)和 strconv.Itoa(2)
    3. 接口：详见”接口的类型断言”

  8. go vet：代码错误检查

  9. 运算符优先级：(.) > (*、&) > (++、--)

    p := &t.x // <=> p := &(t.x) > *p++ // <=> (*p)++

  10. 取余时，余数总是小于被除数，由此可以设置首尾相接的环。

    比如一个 0~3 的环，就可以用 n%4 取余得到，每次 n++后，余数都会变：0，1，2，3，0，1，2，3……

  11. resp.Body.Close():请求响应后应调用该方法，避免内存泄漏

  12. 程序（不包含 Test 程序）的文件层级无论多深，其当前目录始终是项目根目录

  13. cmd 库：

    cmd := exec.Command(“git”, “stash”, “list”)
    err := cmd.Run()        // 无输出运行
    out, err := cmd.CombinedOutput()        // 有输出运行

  14. //go:generate 代码 # 当在终端执行 go generate 时，会执行该代码
    //go:build haha # 写在文件开头，当在终端执行 go build -tags “haha”时，会编译该文件

  15. 不指定类型的常量叫 Untyped 常量,可以作不同类型使用

  16. go build 未指定包或文件时，会编译当前目录
